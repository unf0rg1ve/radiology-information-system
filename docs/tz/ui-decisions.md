## 3. Типографика

- **Основной шрифт:** `'Inter', 'Segoe UI', system-ui, sans-serif`
- **Моноширинный:** `'JetBrains Mono', 'Courier New', monospace` — для AN, ИИН, UID
- **Размеры:** 11px (hint), 12px (таблицы, метки), 13px (основной), 14px (подзаголовки), 16px (h1)
- **Межстрочный интервал:** 1.5 для текста, 1.2 для компактных таблиц

---

## 4. Компоненты

### Бейджи статусов
- Скруглённые пилюли, padding 2px 8px, font-size 11px, font-weight 600
- Перед текстом — цветная точка ●
- Tooltip с кодом API при наведении

### Таблицы
- Zebra-striping (чётные строки — surface2)
- Hover: лёгкая подсветка accent/10%
- Sticky header при скролле
- Строки cito — bg-cito (#fde8e8 / #450a0a)
- Строки срочных — bg-urgent (#fef7e0 / #451a03)

### Формы
- Label сверху, input снизу
- Обязательные поля: красная звёздочка `*`
- Hint text: font-size 11px, text-muted, ниже поля
- Ошибки: красная рамка + текст ошибки под полем
- Readonly поля: surface2, cursor not-allowed

### Навигация
- Сайдбар: фиксированный 200px, тёмный (`--c-sidebar`)
- Активный пункт: левый border-left 3px accent + bg accent/15%
- Счётчик уведомлений: красный бейдж справа
- Топбар: фиксированный, 38px, для переключения экранов в wireframe-режиме

### Кнопки
- Primary: accent bg, white text
- Default: surface bg, border, text
- Danger: danger bg, white text
- Success: success bg, white text
- Размеры: sm (padding 3px 8px), md (5px 12px), lg (7px 16px)

## 6. UX-решения

| Решение | Обоснование |
|---|---|
| Сайдбар тёмный, контент светлый | Чёткое разделение навигации и рабочей зоны |
| Статусы — цветные бейджи с точкой | Быстрое визуальное сканирование worklist |
| cito — красный фон всей строки | Критичность нельзя пропустить |
| 3-колонный экран описания | Врач видит данные пациента, снимки и форму одновременно — нет переключений |
| Вьюер Orthanc встроен (iframe) | Нет перехода на отдельную вкладку; контекст исследования сохраняется |
| Calendar — слот подсвечен при записи | Пользователь видит, на какой слот ведётся запись |
| Автовыбор шаблона протокола по модальности | Снижает ручной ввод, ускоряет описание |
| Инлайн QC-форма в worklist | Не нужен модал; лаборант не теряет контекст списка |
| Генерация AN скрыта от пользователя | Регистратор не вводит AN вручную — только видит результат |
| ИИН: live-валидация + check digit | Ошибки вводаловятся до сохранения, не после |
| Телефон: маска ввода +7 (7__) ___-__-__ | Стандартный формат КЗ, меньше ошибок |

## 7. Справочники — экраны

| Справочник | Действия пользователя | Редактируемость |
|---|---|---|
| Тарификатор МЗ РК | Просмотр, импорт CSV/Excel, ручное добавление/редактирование, версионирование | Администратор |
| МКБ-10 | Только поиск и просмотр (поиск по коду, по названию RU/KZ) | Read-only (импорт системный) |
| Аппараты/модальности | CRUD: добавить, редактировать, изменить статус, расписание | Администратор |
| Шаблоны протоколов | CRUD + конструктор полей + версионирование | Администратор |
| Пользователи и роли | CRUD + смена пароля + блокировка + журнал входов | Администратор |
| Льготные категории | Просмотр + редактирование (ГОБМП, ОСМС, инвалидность и др.) | Администратор |
